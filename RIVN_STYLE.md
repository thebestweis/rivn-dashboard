# RIVN Style

RIVN Style — визуальный язык RIVN OS для рабочих SaaS-интерфейсов. Он должен ощущаться как премиальный, тёмный, живой dashboard: технологичный, чистый, дорогой, но не холодный.

## Общее ощущение

- Тёмная премиальная основа, без плоской “админки”.
- Много воздуха внутри крупных блоков, но без пустоты: каждый блок должен иметь понятный смысл.
- Стекло, мягкие свечения, тонкие контуры, глубокие тени.
- Акцент на удовольствие от взаимодействия: hover, мягкие движения, подсказки, мгновенная реакция.
- Никаких “ИИшных” случайных карточек, разрозненных элементов и кислотного хаоса.

## Палитра

Основной фон:
- `#07111f`
- `#0a1525`
- `#101827`

Карточки:
- `rgba(255,255,255,0.045)` поверх тёмного градиента
- тёмные градиенты с зелёным, фиолетовым или оранжевым оттенком

Главный зелёный:
- `#00f5a8`
- `#43ffc2`
- `#02c98b`

Фиолетовый:
- `#7c5cff`
- `#a78bfa`
- `#5b45d9`

Оранжевый акцент:
- `#ffbf69`

Текст:
- основной: `#ffffff`
- вторичный: `rgba(255,255,255,0.45-0.62)`
- очень тихий: `rgba(255,255,255,0.32-0.42)`

## Фоны

Страница:
- глубокий тёмный фон;
- 2-3 больших размытых radial-gradient пятна;
- аккуратная сетка `84px`, opacity около `0.35-0.45`;
- не использовать декоративные “орбы” как отдельные случайные элементы, только фоновые световые массы.

Основной рабочий контейнер:
- большой radius `40-46px`;
- полупрозрачный тёмный слой;
- border `rgba(255,255,255,0.10-0.14)`;
- blur/backdrop;
- внутренняя верхняя подсветка.

## Карточки

Стандартная карточка:
- radius `28-34px`;
- border `rgba(255,255,255,0.10-0.14)` или цветной border `rgba(accent, 0.16-0.24)`;
- фон: тёмный diagonal gradient с лёгким цветным оттенком;
- shadow: большая мягкая тень + `inset 0 1px 0 rgba(255,255,255,0.12)`;
- padding обычно `24px`;
- заголовок `24px`, medium, tight tracking.

Карточка не должна выглядеть как обычный прямоугольник Tailwind. В ней должен быть слой глубины: градиент, контур, подсветка или стеклянный эффект.

## Стекло

Glass-панели:
- `backdrop-blur: 24-32px`;
- `background: linear-gradient(135deg, rgba(255,255,255,0.34), rgba(255,255,255,0.14), rgba(0,245,168,0.16))`;
- border `rgba(255,255,255,0.20-0.28)`;
- внутри можно использовать мягкий верхний blur-gradient;
- текст на стекле должен иметь хороший контраст, не тонуть в декоре.

## Типографика

- Заголовки карточек: `text-2xl`, `font-medium`, `tracking-[-0.04em]`.
- Большие цифры: `text-5xl`, `font-light`, `tracking-[-0.07em]`.
- Подписи: `text-sm` или `text-xs`, `text-white/45`.
- Не использовать гигантские hero-шрифты внутри dashboard.
- Внутри компактных блоков не перегружать текстом.

## Графики

Финансовая динамика:
- ломаные линии с острыми углами, как в premium analytics;
- зелёная линия для выручки;
- фиолетовая линия для прибыли;
- диагональная штриховка между линиями допустима;
- ось X должна прореживать подписи, если точек много;
- hover должен срабатывать по точкам графика, не по оси X;
- подсказка: тёмное стекло, дата/период, показатель, сумма.

Малые графики:
- точки должны совпадать с линией;
- можно использовать слегка округлённые стыки, но без чрезмерного сглаживания;
- не оставлять плавающие плашки без понятного смысла.

## Показатели

Цвет показателя может отражать динамику периода к периоду:
- рост или отсутствие падения: зелёный;
- падение: фиолетовый;
- серые блоки показывают процентное отклонение и подпись `рост` / `падение`.

В цветном блоке пишем значение.
В сером блоке пишем процент отклонения.

## Интерактивность

- Hover должен быть заметным, но дорогим: лёгкое поднятие, подсветка border, усиление shadow.
- Для абстрактных элементов допустимо движение в сторону курсора.
- Настройки открываются в тёмных стеклянных dropdown, поверх всех слоёв.
- Кнопки должны реагировать мгновенно.
- Не использовать системные dropdown/date-picker, если рядом уже кастомный RIVN-интерфейс.

## Кнопки

Основная:
- pill shape;
- зелёный фон `#00f5a8`;
- текст `#06101d`;
- мягкая зелёная тень.

Вторичная:
- `bg-white/10`;
- border/ring `white/10`;
- текст `white/80`;
- hover `bg-white/16`.

Icon button:
- круг;
- `44-56px`;
- `bg-white/9` или `#08111f`;
- hover зелёный для ключевых действий.

## Абстракции

Фирменные “пузырьки”:
- 3D particle/molecule stream;
- цвета: приглушённый зелёный/бирюзовый + точечный фиолетовый;
- radial-gradient для объёма;
- много шаров разного размера;
- композиция плотная, но не закрывает смысловой текст;
- движение по курсору допустимо.

Абстракция не должна быть кислотной. Цвета должны жить внутри палитры страницы.

## Layout

- Для dashboard использовать крупные сетки с 2 строками и 3 колонками на десктопе.
- Одна выразительная карточка может занимать 2 строки.
- На мобильном карточки складываются в одну колонку, без горизонтального скролла.
- Sidebar проекта пока сохраняет свою структуру, но новые страницы должны визуально дружить с dashboard.

## Чего избегать

- Светлые карточки внутри тёмного RIVN dashboard без адаптации.
- Случайные градиенты и несвязанные цвета.
- Плашки, которые “висят в воздухе” и непонятно что обозначают.
- Перегруженные подписи осей.
- Системные элементы браузера.
- Слишком много текста в карточках.
- Кислотный зелёный без приглушения окружением.
- Колхозная разрозненность: каждый блок должен быть частью одной сетки и одного материала.

## Быстрая команда

Если звучит задача “сделай в RIVN Style” или “переведи страницу в RIVN Style”, нужно:

1. Перевести фон в тёмную премиальную основу.
2. Пересобрать блоки в крупные стеклянные/градиентные карточки.
3. Использовать зелёный, фиолетовый и точечный оранжевый акценты.
4. Сделать заголовки, цифры, графики и hover по правилам выше.
5. Убрать системность, случайность и лишний текст.
6. Проверить адаптивность, контраст и отсутствие наложений.
